# FlashPoint

P2P file transfer in the browser via WebRTC. No servers. No uploads. Purely peer-to-peer.

## Stack

- React 18 + Vite
- PeerJS 1.5 (WebRTC abstraction)
- Pure CSS (OLED dark theme)

## Setup

```bash
npm install
npm run dev
```

Open `http://localhost:5173` in two browser tabs or on two devices on the same network.

## How It Works

1. Both peers open the app and get unique IDs (generated by PeerJS signaling server)
2. The sender pastes the receiver's ID and clicks Connect
3. A WebRTC DataChannel is established directly between the two browsers
4. The sender selects a file — it is sliced into 64 KB chunks and streamed across the DataChannel
5. The receiver reassembles the chunks into a Blob and offers a download link
6. The PeerJS signaling server is only used for the initial handshake — all file data flows peer-to-peer

## Chunking

RTCDataChannel has a maximum message size limit that varies by browser (16 KB – 256 KB). FlashPoint slices every file into 64 KB chunks before sending, ensuring compatibility across all browsers and file sizes.

## Security

- Files never touch a server
- The PeerJS default signaling server (`0.peerjs.com`) is used only to exchange WebRTC ICE candidates
- For production use, deploy your own PeerServer: https://github.com/peers/peerjs-server
